1、Spring-Boot默认使用的什么日志呢？

	由于各种框架和配置集成插件等等默认使用的日志都不一样，但Spring-Boot默认采用的是slf4j加上logback来进行记录，
	所以他将其他的日志文件都进行了转换，全部都转换成slf4j 

2、如何使用默认日志呢？

   1：）首先先打印日志

    //SLF4J日志记录器
    Logger logger = LoggerFactory.getLogger(getClass());

    @Test
    public void contextLoads() {

        //trace < debug < info < warn < error
        //由低到高，可以输出的日志的级别，如果是error级别则只会输出error
        //spring-boot默认给我们设置的级别是info，所以不会打印debug和trace

        logger.trace("这是trcae日志");
        logger.debug("这是DEBUG日志");
        logger.info("这是Info日志");
        logger.warn("这是warn日志");
        logger.error("这是error日志");
    }



    2：）然后我们在配置文件中设置文件属性



logging:
  level:
    #设置test.as下面的所有的目录的日志级别为trace，如果不设置默认为info级别
    test.as: trace
  #设置日志文件路径，如果不设置绝对路径则会在当前的项目中的根目录下生成文件，
  #如果设置了绝对路径则会在硬盘制定的目录下输出路径
  #file: spring-boot.log
  #如果设置了这个文件路径的话他会在当前的磁盘的根目录下的/spring/log文件夹下生成一个叫做spring.log的文件
  #注意如果设置这个方式添加日志，那么就要把上面的file删掉否则他将不会执行
  path: /spring/log
  pattern:
    #在控制台中输出的日志的格式
    console:  "%d{yyyy-MM-dd HH:mm} --- %-5level %logger{50} - %msg%n"
    #在文件中输出的日志的格式
    file: "%d{yyyy-MM-dd} --- === %-5level === %logger{50} - %msg%n"
